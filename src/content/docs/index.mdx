## Building got cheap. Judgement got scarce.

An agent can draft the spec, write the code, and run the tests before you've
booked the kickoff. The bottleneck moved. It isn't capacity any more. It's
knowing whether the thing is right, and catching it when it isn't.

ProductOS keeps judgment in the loop while everything underneath runs at agent speed.

## One rule decides what ships

<Aside type="tip" title="The verdict rule">
A change ships only when it advances a vision outcome, satisfies its Job Spec,
passes every principle, and crosses no invariant.
</Aside>

Agents build fast inside those gates. You own the gates.
